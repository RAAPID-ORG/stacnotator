# Contributing

We are happy to see your are interested in contributing to this project. We are happy to integrate any contributions that benefit the wider geospatial community.

Especially we welcome:
- Bug reports
- Feature requests and proposals with justifaction and examples opened as issues
- Pull requests for specific features

## How to contribute a feature

Make sure to familiarize yourself with the codebase by checking out the [docs](docs/architcture.md).

We reccomend only implementing a completely new feature after opening a issue first, to ensure it align with the core teams long term vision that benefits the community. We strive to add functionality that provides core functionality without bloating the project for specific use cases.


**Opening a PR**
Please select `develop` as the target branch. All changes are first tested in a deployed dev version before making it into the final release.

- Ensure your code is well tested if applicable and provide a short description of what you are doing.
- For contributions that rely on LLMs, please ensure your code is not testing mock functionality, but rather core business logic.
- Code should be mostly self documenting.

We will review the PR as soon as capacities free up, which might take a few days.
